# AI and Copilot

A cluster of `.client` constructs for IFS Cloud's AI/Copilot features: page-level AI disclaimers and settings, Copilot chat prompts, and an AI-assisted rich text editor (`smarteditor`) with its own AI-triggered commands and input parameters.

---

## `aisettings`

> [!abstract] Syntax
> ```marble
> aisettings {
>    indicateai = [<condition>];
>    [disclaimer { text = "<Value>"; }]
> }
> ```

"Defines AI related settings or features." Can appear on a `page`, `dialog`, `assistant`, `group`, `command`, or `card`.

```marble
aisettings {
   indicateai = [true];
}
```

### `disclaimer`

Nests inside `aisettings`. "Defines AI disclaimer properties."

```marble
disclaimer {
   text = "This content was generated by AI and may be inaccurate.";
}
```

---

## `copilotprompt`

> [!abstract] Syntax
> ```marble
> copilotprompt {
>    description = "<Value>";
>    prompttext  = "<Value, can include ${AttrRef} interpolation>";
> }
> ```

"Defines a page level prompt that will appear in the Copilot chat."

```marble
copilotprompt {
   description = "List failure modes for this FMECA";
   prompttext = "Please provide me with the most applicable Failure Modes for ${ProcessClassIdRef.ProcessClassDesc} and ${ItemClassIdRef.ItemClassDesc}";
}
```

Note the `${AttrRef.NestedAttr}` interpolation syntax inside `prompttext` — it pulls live field values from the page into the prompt sent to Copilot.

## `smarteditor`

> [!abstract] Syntax
> ```marble
> smarteditor {
>    enabled = [<condition>];
>    [smart command { ... }]   -- one or more
>    [improve command ...]     -- see note below
> }
> ```

"A control which enables users to enhance and format the content using AI features." Attaches to a `field` reference (a rich-text field, typically).

```marble
smarteditor {
   enabled = [true];
}
```

> [!info] `ImproveCommandDefinition` not covered here
> `smarteditor`'s body can also contain an `improve` command — that construct already had some vault coverage before this pass and wasn't part of this gap-filling round, so it isn't documented in depth here. It sits alongside `smart command` as another AI-triggered action inside the editor.

### `smart command`

Nests inside `smarteditor`. "Defines smart editor commands — these commands leverage AI features to execute their intended functionality."

```marble
smarteditor {
   smart command {
      label = "Generate Description with AI";
      enabled = [true];
      execute {
         call GenerateSmartObjective();
      }
   }
}
```

A `smart command` can optionally reference a `smartinputs` block via `SmartInputReference` to pass structured parameters into the generation call.

---

## `smartinputs`

> [!abstract] Syntax
> ```marble
> smartinputs <Name> for <Datasource> {
>    field <FieldName>;
>    -- one or more fields, used as content-generation input parameters
> }
> ```

"A component used to define input parameters used for content generation in smart editor." Standalone (top-level) — not nested inside `smarteditor` itself, but referenced from a `smart command`.

```marble
smartinputs SampleSmartInputs for InputParameterStructure {
   field Prompt;
   field Tone;
}
```
